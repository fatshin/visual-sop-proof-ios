Agent Autopsy starts with a failed support-agent trace. The trace silently changes the customer identity, issues a refund above the autonomous limit, and repeats the same side effect. Running the autopsy isolates all three failures, creates one regression assertion per failure, and proposes only the narrow patch needed for each cause. The system does not execute a patch or replay a production transaction. I used Codex with GPT-5.6 to design the failure representation, implement causal checks, write the regressions, and challenge the patch scope. A live GPT-5.6 extension can propose additional hypotheses, while trace evidence and deterministic checks decide whether a failure is confirmed.
