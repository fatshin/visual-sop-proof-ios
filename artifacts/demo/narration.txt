Agent Autopsy starts with a tested failed support-agent trace. The trace silently changes the customer identity, issues a refund above the autonomous limit, and repeats the same side effect. Deterministic checks identify all three supported failure modes, create one regression assertion per finding, and propose a scoped patch suggestion for human review. The system does not infer a complete root cause, execute a patch, or replay a production transaction. I used Codex with GPT-5.6 to design the failure representation, implement the explicit checks, write the regressions, and challenge the proposed patch scope. A live GPT-5.6 extension can propose additional hypotheses, while trace evidence and deterministic checks decide whether a supported failure mode is confirmed.
