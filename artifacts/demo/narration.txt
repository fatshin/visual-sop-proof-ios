Agent Autopsy starts with a failed support-agent trace. The trace silently changes the customer identity, issues a refund above the autonomous limit, and repeats the same side effect. Running the autopsy isolates all three failures, creates one regression assertion per failure, and proposes only the narrow patch needed for each cause. The system does not execute a patch or replay a production transaction. GPT-5.6 can generate additional hypotheses, while trace evidence and deterministic checks decide whether a failure is confirmed.
