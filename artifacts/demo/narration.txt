Meeting to Merge preserves the path from a spoken requirement to a proposed code change. The fixture contains a complete, uniquely identified set of four timestamped requirements and an explicit definition of approval. Running the analysis reproduces four baseline failures, maps four deterministic post-patch scenarios, and produces a minimal unified diff covering input validation, approval, receipt identity, and idempotency. The generated source is parsed as Python and rejected if any function call appears before the non-empty order-ID guard. The demo does not execute or apply the patch. I used Codex with GPT-5.6 to translate intent into checks, regression-test the CRLF fix, and challenge incomplete, duplicated, and unsupported requirements. The patch stays behind a human apply boundary.
