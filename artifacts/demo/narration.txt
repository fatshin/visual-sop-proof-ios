CostRoute Lab uses measured fixture quality and normalized cost rather than intuition to choose a model route. Exactly twenty case IDs each map to one task, quality stays between zero and one hundred, and every case-task group contains one row per model. The GPT-5.6 baseline bundle cost must be positive before any reduction is calculated. With the macro-average floor set to ninety-one, exhaustive search finds a route scoring ninety-one point five with a normalized bundle cost of three point zero. That is a sixty-one percent fixture reduction against the baseline cost of seven point seven. Missing cases, duplicate rows, invalid quality, missing models, or an unreachable floor fail closed. I used Codex with GPT-5.6 to implement exhaustive routing and review every unit and aggregation claim. These numbers are synthetic demo observations, not current billing quotes.
